# 兵无常势（FluidWarfare）战斗结算原型 — 设计任务

## 项目背景

我正在开发一款战术推演游戏《兵无常势》。当前阶段是**战斗结算原型的算法验证**——用一个 Electron 原型（代号"奇正相生"）试验六力模型和训练模板系统是否合理，再应用到完整游戏。

完整游戏的设计总纲：
- 全尺度动态统帅模拟器：从师级战役到营级微操
- 视觉上采用语义缩放（MIL-STD-2525 北约符号 → 定制单兵 UI → 2D Spine 动画）
- 核心玩法：宏观 AI 托管 + 关键点玩家微操介入（莫德尔式"救火队长"）
- 微操下限 = 营，危急时可沉入连
- 对标竞品：钢铁雄心 4，但用空间几何和战术逻辑替代数值对撞

原型验证范围：单兵 → 班 → 排 → 连 → 营，逐层跑通六力模型。

## 已完成的设计（v3，已写入 doc/six-forces-design.md）

### 六力模型

六大作战能力维度：打击力、生存力、机动力、感知力、控制力、保障力。

### 三状态基座

三个 0~100 的状态值替代传统 HP：

| 状态 | 含义 | 输入来源 | 影响 |
|------|------|---------|------|
| 体能 | 肌肉+代谢+水合 | 口粮 + 训练水平 + 负重 | 移速、消耗率、射速 |
| 士气 | 凝聚力+求生本能 | 战斗经验 + 伤亡冲击 + 口粮 | 崩溃阈值、命令服从 |
| 专注 | 认知带宽 | 战斗经验 + 被压制程度 | 精度、发现概率 |

**口粮滑块（800~4500 kcal/天）驱动体能和士气的恢复/衰减：**
- 体能恢复 = (口粮/3500) × 基准恢复率，上限 1.2
- 士气恢复 = (口粮/3500) × 基准恢复率，上限 1.0
- 口粮 < 2500：体能每日衰减
- 口粮 < 1800：士气每日衰减
- 口粮 < 1000：求生本能触发概率 +30%

**状态联动：**
- 体能 < 30% → 士气消耗 ×1.5
- 士气 < 20% → 专注上限 ×0.5
- 士气归零 → 求生本能接管，拒绝攻击命令

### 武器物理推导（4 输入 → 4 输出）

输入：caliber（口径 mm）、action（bolt/semi/auto）、barrelLength（枪管长 mm）、sightMag（瞄具倍率）

```
精度 = 0.4 + barrelLength/2000 + (sightMag-1)×0.05
射程(m) = caliber × barrelLength / 7
射速(ms) = 60000 / actionRPM        (bolt=15, semi=40, auto=120)
致命性 = caliber × actionKillMod    (暂定 1.0)
```

### 无 HP 伤势系统

```
命中后掷 1d10:
  d10 ≤ 致命性−4 → 致命（永久失去战斗力）
  d10 ≤ 致命性    → 重伤（移速×0.3, 精度×0.4, 禁射3s, 15s后恢复为轻伤）
  其余            → 轻伤（移速×0.5, 精度×0.7, 5s后恢复）
  叠加: 3轻→重伤, 2重→致命
```

距离衰减：
```
ratio = distance/effectiveRange
≤0.5: 命中修正=1.0
0.5~1.0: =1.0−(ratio−0.5)×0.4
>1.0: =max(0, 0.8−(ratio−1.0)×1.6)
```

## 正在设计中的部分（训练模板系统）

### 核心问题

六力公式的输入变量（体能基线 70 vs 50、士气基线 75 vs 30）目前仍是主观赋值。训练模板系统的目标是**让这些基线值从客观的训练配置中推导出来**——类比 HOI4 师编制设计器，玩家组合训练科目形成训练大纲，科目累加推演出六力。

### 已确定的方向

**1. 训练模板粒度 = 班（Squad）**
- 班是最小同质训练单元（一个班只干一件事）
- 一排内所有班同兵种、共用一套训练模板
- 不同排可用不同模板，不同兵种的排合成连
- 战时微操下限 = 营，训练模板在班层定义，战斗中自动聚合到营

**2. 结构：六板块 ×（1 必修 + N 自由选修）**
- 六个板块对应六力
- 每板块一门必修课（不可移除，保底），不同兵种必修不同
- 选修无槽数上限

**3. 防止"万能最优模板"的机制**
- 平方根收益递减：`科目效果 = 基础效果 × √(投入周数/标准周数)`
- 必修课随兵种变化（步兵班、机枪班、迫击炮班各自不同必修）→ 不同兵种从起点就分叉
- 偏科通过"专精科目替代基础科目"，不允许同一科目重复叠加

**4. 提前部署（类 HOI4 训练度 20% 可部署）**
- 最低部署门槛 = 目标周数 × 20%
- 两层惩罚：(a) 训练水平 × √完成度（数值打折）；(b) 高阶科目结构性缺失

**5. 民兵/游击队有独立科目池**
- 不是正规军缩水版——必修课不同（政治动员替代基础纪律，游击战术替代战场常识）
- 独有科目：爆破训练、夜间行动、地形利用
- 正面作战弱（打击力/生存力低），特定场景强（士气高、地形特化）

**6. 训练水平推导三状态基线**
```
训练总周数 = Σ(已选科目周数)
训练水平 = clamp(总周数 × 训练强度 × 5, 10, 85)
体能基线 = 训练值 × 0.7 + 30
士气基线 = 训练值 × 0.5 + 纪律加成 + 战斗经验修正
专注基线 = 训练值 × 0.4 + 30 + 战斗经验修正
```

## 希望得到的帮助

以下开放性问题，请选择你最有想法的方向给出建议：

### A. 训练模板系统
1. 目前的六板块结构是否合理？有无更好的组织方式？
2. 必修+自由选修的结构有没有潜在的设计漏洞（如"最优解"仍可被算出）？
3. 科目跨板块的次要效果（如负重行军→主效果机动力，次效果打击力精度+0.03）是否应该纳入？
4. 你有什么科目设计的建议？特别是不同兵种的区分度方面。

### B. 六力模型本身
1. 六力评价一支军队是否合理？有没有遗漏的维度或多余的维度？
2. 三状态（体能/士气/专注）和六力之间的交叉影响是否有双重计算的风险？
3. 对于营级聚合（从班→排→连→营），各力应该如何聚合？取均值、取最差值、还是加权？

### C. 战斗结算
1. 1d10 致命性判定机制是否足够好？有无更优的替代方案？
2. 压制力（见文档第七章）的设计是否合理？
3. 恐慌传染（50m 内士气-10）在营级尺度下如何调整？

### D. 整体评价
1. 这个六力+"训练模板推导"的方案相比传统兵棋的"数值黑盒"（如 HOI4 的软攻/硬攻/组织度），优势和劣势分别是什么？
2. 原型验证阶段应该优先验证什么？有没有我们忽略的关键验证点？

## 约束条件
- 拒绝主观赋值——所有数值必须有客观来源（物理量或可训练属性）
- 保持公式简单——单兵层面可手算验证
- 尺度无关——同一套公式从单兵到营都适用，只是颗粒度注释不同
- 原型阶段不追求完美，暴露问题 > 解决问题
