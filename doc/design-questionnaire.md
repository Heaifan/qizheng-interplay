# 兵无常势（FluidWarfare）设计问卷

我正在设计一款战术推演游戏《兵无常势》的战斗结算原型。以下是我在设计过程中反复纠结的核心问题，希望得到你的独立判断和推理过程，而非迎合我的已有倾向。

---

## 背景速览

- 游戏尺度：玩家作为全国统帅，可缩放至营级微操（对标 HOI4 但允许关键点手动干预）
- 战斗结算：六力模型（打击力/生存力/机动力/感知力/控制力/保障力）+ 三状态基座（体能/士气/专注）+ 无 HP 骰子伤势系统
- 训练系统：玩家组合训练科目定义部队训练大纲 → 推导六力基线值（类比 HOI4 师编制设计器）
- 原型阶段：单兵 → 班 → 排 → 连 → 营，逐层验证算法

---

## 问题一：微操下限定在哪一层？

**背景：**

游戏允许玩家从全国统帅缩放至战术层面。我在连、排、营之间犹豫了很久。

| 选项 | 一个师下属量 | 理由 |
|------|------------|------|
| 连 | ~30 个棋子 | 是复合单位（步兵排+机枪排+迫击炮排），战术灵活 |
| 排 | ~100 个棋子 | 同质单位（一个排只干一件事），命令清晰 |
| 营 | ~9 个棋子 | 最小合成战斗单元（自带迫击炮/反坦克/机枪支援），独立破局 |

我最初倾向连——因为钢铁雄心 4 太"宏观旁观"了，我希望玩家能微操到连级获得参与感。

但被其他 AI 质疑：巴巴罗萨 150 个师 ≈ 4000+ 个连，玩家大脑无法处理。而且历史上连长关心的不是"单兵训练大纲"，而是"这个连能不能完成任务"。

目前暂定：**平时管营，急了管连**。营是默认微操下限，连是紧急干预时的"打破层级"行为。

**请回答：**

1. 你认同"营为主、连为紧急"的方案吗？还是你认为应该固定在一个层级不变？
2. 如果固定在一个层级，你认为应该是哪一层？为什么？
3. 微操下限的选择会如何反过来影响训练模板和六力模型的设计？

## 问题二：训练模板以班为单位是否可行？

**背景：**

训练模板 = 玩家组合训练科目形成的"出厂配置单"。我计划将其放在**班（Squad）**层——班是最小同质训练单元（一个班只干一件事，如步兵班、机枪班、迫击炮班）。

反对意见：
- "师长不关心单兵训练大纲"
- 如果模板在班层，玩家可能把不同职能的班拼成一个"超人排"——反步+反坦+防空+狙击+特战+装甲全塞进去
- 班层定制太碎，玩家管不过来

我的防御设计：
- **一排内所有班必须同兵种、共用同一训练模板**——堵死混搭
- 不同排可用不同模板 → 不同排组成合成连
- 战时玩家只看营级棋子，训练模板是战前编辑器的活

但仍有疑虑：玩家在编辑器里设计班的训练模板 → 组装排 → 组装连 → 赋予营，这个链条是不是太长了？

**请回答：**

1. 训练模板放在班层，你认为合理吗？如果不合理，应该放在哪一层（排？连？营？）？
2. "一排同模板"这个约束是否足够阻止"高达师"？有没有其他漏洞？
3. 从玩家体验角度，你会在意"编辑器的粒度比战场微操细两级"这件事吗？

## 问题三：训练模板的结构怎么设计？

**背景：**

当前方案：**六个板块（对应六力）×（1 门必修 + N 门自由选修）**。

- 必修课随兵种变化（步兵班必修"基础射击"，机枪班必修"机枪操作"）
- 选修无槽数上限，但平方根收益递减：`科目效果 = 基础效果 × √(投入周数/标准周数)`
- 跨专业惩罚：`科目有效率 = 1/√(科目类别数量)`——学太多种类的科目，每样都打折
- 玩家自由设定目标训练周数，最低 20% 可提前部署（类 HOI4），代价是训练水平 × √完成度 + 高阶科目缺失

**请回答：**

1. 六个板块（对应六力）的组织方式是否自然？如果你来设计，你会按什么维度组织训练科目？
2. 必修 + 自由选修 vs 全部自由选修（纯 HOI4 式），你倾向哪个？为什么？
3. 平方根衰减 + 跨专业惩罚，这两个机制是否足够阻止"最优模板"？还有没有漏洞？
4. 民兵/游击队应该用同一套系统（只是可选科目少），还是用完全不同的科目池？你的理由是什么？

## 问题四：六力模型的完整性和正确性

**背景：**

六力 = 打击力、生存力、机动力、感知力、控制力、保障力。

这三个维度和中美军事学说（美军 ADP 3-0、解放军军语）的作战要素基本重合。但有人提出：
- 保障力太薄（目前只有携弹量+补给速率，缺少医疗后送、工事构筑）
- 可能缺"工程/工事"维度（突破雷场、炸碉堡、架浮桥）
- 三状态（体能/士气/专注）和六力之间可能存在双重计算（训练既提升体能基线，训练科目又直接加机动力）

**请回答：**

1. 六力是否有遗漏或多余的维度？如果让你增删一个，你会怎么改？
2. 保障力应该如何扩展才不变成"弹药计数器"？
3. 如何避免三状态和六力之间的双重计算？你会怎么设计它们的分层关系？

## 问题五：班→排→连→营的聚合规则

**背景：**

当前方案是"同样公式，只改变颗粒度注释"——从单兵到营全用同一套六力公式。

但聚合方式尚未确定。之前讨论过一个方向：

| 力 | 聚合方式 | 理由 |
|------|---------|------|
| 打击力 | 加总后乘协同系数 | 火力可叠加，但受控制力限制 |
| 生存力 | 加权均值+最差班惩罚 | 短板会拖累整体 |
| 机动力 | 取 P25（较慢的四分之一） | 全营受慢单位拖累 |
| 感知力 | 取最佳值×信息共享效率 | 侦察班发现后能共享 |
| 控制力 | 乘法（营×连×排×班） | 指挥链是串联系统 |
| 保障力 | 加总资源池 | 弹药/医疗共享 |

**请回答：**

1. 这个按力分别聚合的方案是否合理？有没有你觉得应该改成别的聚合方式的？
2. 控制力用乘法意味着一个烂环节拖垮整条链——这在现实中成立，但在游戏中会不会太苛刻？
3. 如果聚合后只剩一个平均战斗力数字，这个系统就失败了。你觉得怎样才算"聚合成功"？

## 问题六：骰子系统的选择

**背景：**

当前伤势判定用 1d10：

```
命中后掷 1d10:
  d10 ≤ 致命性−4 → 致命
  d10 ≤ 致命性    → 重伤
  其余            → 轻伤
```

有人建议用 2d6（钟形曲线，中间结果更常见，极端结果更少见），认为更"物理真实"。

我的顾虑：1d10 玩家能心算概率（致命性 8 → 致命 40%/重伤 40%/轻伤 20%，一眼看懂），2d6 需要查表。

**请回答：**

1. 对于这个原型，1d10 和 2d6 你选哪个？理由是什么？
2. "心算友好" vs "概率分布精度"，在原型阶段你更看重哪个？
3. 如果保留 1d10，需要在什么环节补偿它的"均匀分布"劣势吗？如果不需要，为什么？

## 问题七：宏观 AI 托管 + 玩家微操介入

**背景：**

核心玩法：战线由 AI 托管推进，玩家在关键点（一个车站、一座桥、一个山头）沉入微操 5 分钟，通过战术机动打破 AI 僵局，然后回到宏观视角看战线因这个缺口而松动。

这和训练模板系统的关系：大部分时间部队在 AI 手里。训练不只应该给数值加成，还应该决定 AI 托管时部队的"战术性格"——精锐自动找掩体、交替掩护；民兵可能扎堆、乱跑。

**请回答：**

1. 你认同"训练决定 AI 托管时的行为"这个方向吗？如果不认同，你觉得 AI 行为应该由什么决定？
2. "战术性格"用十几个行为标签（如"瞄准后再开火"、"被压制时卧倒"、"移动时交替掩护"）来实现，是合理的抽象层级还是过度工程化？
3. 玩家微操介入的 5 分钟里，你觉得什么体验最重要？是"看到自己的战术决策生效"还是"看到自己的部队因训练有素而执行得力"？
